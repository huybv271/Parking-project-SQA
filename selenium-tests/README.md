# Parking Management System - Selenium Test Automation

Selenium-based browser automation tests for the LICHT-PARKING management system.

## Project Structure

```
selenium-tests/
├── test_cases/               # Test configuration and data
│   └── selenium_test_generation_config.json
├── tests/                    # Test modules
│   ├── conftest.py          # Pytest configuration & fixtures
│   ├── test_auth.py         # Authentication tests (staff/customer login)
│   ├── test_checkin.py      # Check-in/ticket creation tests
│   └── test_checkout.py     # Check-out tests
├── pages/                    # Page Object Model classes
│   ├── base_page.py         # Base page class with common methods
│   ├── login_page.py        # Login page objects (staff & customer)
│   ├── checkin_page.py      # Check-in page object
│   └── checkout_page.py     # Check-out page object
├── utils/                    # Utility modules
│   ├── config.py            # Configuration & environment variables
│   ├── driver_factory.py    # WebDriver setup & management
│   └── screenshot_helper.py # Screenshot capture on failure
├── screenshots/             # Auto-generated screenshots (test failures)
├── reports/                 # Auto-generated HTML test reports
├── pytest.ini              # Pytest configuration
├── requirements.txt        # Python dependencies
└── README.md               # This file
```

## Setup Instructions

### Prerequisites

- Python 3.8 or higher
- Google Chrome browser (driver auto-managed by webdriver-manager)
- Application running on `http://localhost:3000`

### Quick Start (Windows PowerShell)

```powershell
cd selenium-tests
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**Note:** If PowerShell execution policy error occurs, run:
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 1. Create Virtual Environment

#### Windows PowerShell
```powershell
cd selenium-tests
python -m venv venv
venv\Scripts\Activate.ps1
```

#### Windows Command Prompt (CMD)
```cmd
cd selenium-tests
python -m venv venv
venv\Scripts\activate.bat
```

#### macOS/Linux
```bash
cd selenium-tests
python -m venv venv
source venv/bin/activate
```

### 2. Install Dependencies

```powershell
pip install -r requirements.txt
```

This installs:
- **selenium** (4.15.2): Browser automation
- **pytest** (7.4.3): Test runner
- **webdriver-manager** (4.0.1): Chrome driver management
- **pytest-html** (4.1.1): HTML report generation
- **python-dotenv** (1.0.0): Environment variable management
- **openpyxl** (3.11.0): Excel file support (optional)

### 3. Configure Environment

Create a `.env` file in the `selenium-tests/` directory with your credentials:

```env
# Application URL
BASE_URL=http://localhost:3000

# Staff/Admin Credentials
STAFF_USERNAME=staff1
STAFF_PASSWORD=password123
ADMIN_USERNAME=admin1
ADMIN_PASSWORD=admin123

# Customer Credentials
CUSTOMER_USERNAME=customer1
CUSTOMER_PASSWORD=password123

# Browser Settings
HEADLESS=false
WINDOW_WIDTH=1920
WINDOW_HEIGHT=1080

# Screenshot Settings
SCREENSHOT_ON_FAILURE=true

# Optional: PlateRecognizer API Key (for future camera automation)
PLATE_RECOGNIZER_API_KEY=
```

## Test Cases

### Automated Tests (Working)

#### Authentication Tests (`test_auth.py`)
- **AUTH_STAFF_01**: Staff login with valid credentials
- **AUTH_STAFF_02**: Staff login with invalid password
- **AUTH_STAFF_03**: Staff login with empty fields
- **AUTH_CUSTOMER_01**: Customer login page navigation
- **AUTH_CUSTOMER_02**: Customer login with valid credentials
- **AUTH_UI_01**: Staff login page UI elements visible
- **AUTH_UI_02**: Customer login page UI elements visible

### Skipped Tests (Require Camera/OCR)

#### Check-in Tests (`test_checkin.py`) - All SKIPPED
- **CI_FUN_01** to **CI_FUN_09**: All check-in tests require camera stream access and PlateRecognizer API

#### Check-out Tests (`test_checkout.py`) - All SKIPPED
- **CO_FUN_01** to **CO_FUN_09**: All check-out tests require camera stream and parking session pre-condition

### Why Camera Tests Are Skipped

Selenium cannot reliably automate camera-based tests because:

1. **Browser Camera Access**: Selenium has no built-in way to:
   - Programmatically grant camera permissions
   - Inject camera stream frames
   - Mock `getUserMedia()` API responses

2. **OCR/Plate Recognition**: The backend uses external PlateRecognizer API which:
   - Requires valid image files with readable license plates
   - Needs proper API key and network access
   - Cannot be mocked by Selenium alone

3. **Environmental Dependency**: Test machines need:
   - Compatible camera hardware
   - Camera drivers installed
   - Permission settings compatible with browser automation

### Enabling Camera Tests (Future)

To enable camera-based tests, implement:

1. **Backend Test Mode**:
   - Create test endpoint accepting image uploads
   - Pre-configured test plates and mock responses
   - Ability to seed parking sessions via API

2. **Test Fixtures**:
   - Pre-recorded test images with known license plates
   - Test data organized by vehicle type and plate format

3. **Selenium Grid / Browser Extensions**:
   - Use Selenium Grid with browser extensions for camera control
   - Implement mock camera provider
   - Or use browser-specific automation tools

Example future implementation:
```python
# Helper to create parking session for testing
@pytest.fixture
def test_parking_session():
    response = requests.post(
        "http://localhost:3000/api/test/tickets",
        json={"plate": "59-D2420.69", "vehicle_type": "MOTORBIKE"}
    )
    return response.json()

# Helper to upload test image
def upload_test_checkin_image(plate):
    with open(f"test_images/{plate}.jpg", "rb") as img:
        response = requests.post(
            "http://localhost:3000/staff/ticket-entry",
            files={"image": img}
        )
    return response.json()
```

## Page Object Model

### BasePage (`pages/base_page.py`)
Base class providing common Selenium methods:
- `wait_for_element()`: Wait for element presence
- `wait_for_element_clickable()`: Wait for clickable element
- `click_element()`: Click after waiting
- `fill_input()`: Fill input field
- `get_text()`: Get element text
- `is_element_visible()`: Check visibility
- `navigate_to()`: Navigate to URL
- `screenshot()`: Take screenshot

### LoginPage (`pages/login_page.py`)
- `LoginPage`: Customer login
- `StaffLoginPage`: Staff/Admin login
- Methods: `navigate_to_*()`, `login_*()`, `is_login_successful()`

### CheckinPage (`pages/checkin_page.py`)
- Methods for check-in page interaction
- Camera status checking
- Result extraction (plate, vehicle type, area, position)

### CheckoutPage (`pages/checkout_page.py`)
- Methods for check-out page interaction
- Bill information extraction
- Image verification

## Screenshots


To enable/disable:
```env
SCREENSHOT_ON_FAILURE=true  # or false
```

## Troubleshooting

### Chrome Driver Issues
- **webdriver-manager** automatically downloads the correct Chrome driver
- If issues persist:
  ```powershell
  pip install --upgrade webdriver-manager
  ```

### Element Not Found
- Check if the application is running on `http://localhost:3000`
- Verify selectors in page objects match current HTML
- Take screenshots to inspect page state: `driver.save_screenshot("debug.png")`

### Timeout Errors
- Increase `DEFAULT_WAIT` in `.env`
- Check application response time
- Verify network connectivity

### Camera Permission Denied
- Camera tests are SKIPPED by default (expected behavior)
- See "Enabling Camera Tests" section for future implementation

## CI/CD Integration

For GitHub Actions or similar CI/CD systems:

```yaml
# .github/workflows/selenium-tests.yml
name: Selenium Tests
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-python@v2
        with:
          python-version: '3.10'
      
      - name: Install dependencies
        working-directory: ./selenium-tests
        run: |
          python -m venv venv
          source venv/bin/activate
          pip install -r requirements.txt
      
      - name: Run tests
        working-directory: ./selenium-tests
        run: |
          source venv/bin/activate
          pytest --html=reports/report.html --self-contained-html
      
      - name: Upload report
        if: always()
        uses: actions/upload-artifact@v2
        with:
          name: pytest-report
          path: selenium-tests/reports/
```

## Best Practices

1. **Use Page Objects**: All UI interactions through page objects, not direct driver calls
2. **Explicit Waits**: Use explicit waits instead of `time.sleep()`
3. **Locator Priority**: Use ID → name → data-testid → CSS selector → XPath
4. **Test Independence**: Tests should not depend on execution order
5. **Descriptive Names**: Test names should clearly describe what's tested
6. **Assertions**: Use meaningful assertion messages

## Contributing

When adding new tests:

1. Create tests in appropriate file (`test_auth.py`, `test_checkin.py`, etc.)
2. Use Page Object Model for all UI interactions
3. Add docstring with TC_ID and description
4. Use explicit waits, not `time.sleep()`
5. Take screenshots for manual inspection if needed
6. Add to pytest markers if categorizing
